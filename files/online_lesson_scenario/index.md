# Интеграция Системы А (Электронный дневник) и Системы В (ВКС)

## 1. Пользовательский сценарий проведения онлайн-урока

В данном сценарии инициатором выступает **Преподаватель**, который запускает занятие из интерфейса Системы А.

**Сценарий (Happy Path):**

1. **Инициализация:** Преподаватель заходит в расписание Системы А, выбирает нужный урок и нажимает кнопку «Начать онлайн-урок».
2. **Создание среды:** Система А формирует название чата (например, «Проектирование информационных систем, лекция 5») и отправляет запрос `B.createChat`. Система В возвращает `ID_chat`.
3. **Наполнение среды:** Система А получает из своей базы список идентификаторов (для Системы В) преподавателя и студентов, прикрепленных к этому уроку. Затем поочередно добавляет их в созданный чат с помощью метода `B.addChatUser`.
4. **Создание звонка:** Система А вызывает метод `B.start`, передавая полученный ранее `ID_chat`. 
5. **Подключение:** Система В возвращает `Call_ID` (сохраняется в базе Системы А для последующего управления) и `Call_link` (ссылку на звонок).
6. **Отображение:** Система А делает `Call_link` доступной в интерфейсе: для преподавателя происходит автоматический редирект, а для студентов появляется кнопка «Присоединиться к уроку» в расписании.
7. **Проведение урока:** Пользователи переходят по ссылке и общаются в Системе В.
8. **Завершение:** По окончании урока преподаватель нажимает в Системе А кнопку «Завершить урок» (либо срабатывает таймер расписания). Система А отправляет запрос `B.forceFinish(Call_ID)`. Звонок в Системе В завершается.

## 2. Алгоритм добавления пользователей в чат

Алгоритм описывает логику работы бэкенда Системы А после того, как поступила команда на старт урока. 

### Входные данные для алгоритма:
* `Lesson_ID` — идентификатор текущего урока в Системе А.
* `Teacher_ID` — идентификатор преподавателя в Системе А.

### Шаги алгоритма:

1. **Подготовка данных:**
   * Запросить из БД Системы А данные об уроке по `Lesson_ID` для формирования названия (сохранить в `Chat_name`).
   * Запросить из БД Системы А список участников (студенты + преподаватель) и получить их сопоставленные идентификаторы для Системы В. Записать массив в `Users_B_List`.
2. **Создание чата:**
   * Выполнить API запрос к Системе В: `B.createChat(Chat_name)`.
   * Получить ответ и сохранить идентификатор чата в `ID_chat`.
3. **Цикл добавления пользователей:**
   * Инициализировать цикл, который будет проходить по каждому элементу `Current_User_ID` из массива `Users_B_List`.
   * **Начало цикла:**
     * Выполнить API запрос: `B.addChatUser(ID_chat, Current_User_ID)`.
     * Дождаться ответа от Системы В (в рамках текущей задачи предполагаем, что всегда возвращается "OK", негативные сценарии игнорируем).
   * **Конец цикла.**
4. **Инициация звонка:**
   * Выполнить API запрос: `B.start(ID_chat)`.
   * Сохранить полученные `Call_ID` и `Call_link` для передачи на клиентскую часть Системы А.

### Пример реализации логики (Python)

```python
def start_online_lesson(lesson_id: int) -> str:
    # 1. Подготовка данных
    chat_name = f"Лекция {lesson_id}" # В реальности запрос к БД за названием
    users_b_list = get_system_b_user_ids_for_lesson(lesson_id) # Запрос к БД за ID пользователей

    # 2. Создание чата
    id_chat = B.createChat(Chat_name=chat_name)

    # 3. Цикл добавления участников
    for current_user_id in users_b_list:
        response = B.addChatUser(ID_chat=id_chat, ID_user=current_user_id)
        
        # В рамках данного ТЗ игнорируем обработку response == 'Error'
        if response == "OK":
            continue 

    # 4. Запускаем звонок после того, как все участники добавлены в чат
    call_data = B.start(ID_chat=id_chat)
    
    # Сохраняем call_data.Call_ID в базу данных Системы А для возможности вызова forceFinish
    save_call_id_to_db(lesson_id, call_data.Call_ID)

    # Возвращаем ссылку на фронтенд дневника для редиректа/отображения кнопки
    return call_data.Call_link
