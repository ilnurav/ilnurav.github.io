# Работа 7: Настройка CI/CD в GitLab

**Цель работы:** Познакомиться с концепцией непрерывной интеграции и доставки (CI/CD), изучить основные компоненты GitLab CI/CD и научиться настраивать автоматическое тестирование и "развертывание" простого приложения.

---

## Часть 1. Теоретическое введение (Просто о сложном)

Представьте, что вы строите дом из лего вместе с пятью друзьями. Каждый собирает свою часть. Если вы попытаетесь соединить все части только в самом конце, скорее всего, крыша не подойдет к стенам, а дверь окажется на месте окна. 

Чтобы этого избежать, нужно постоянно примерять новые детали к общему дому. Это и есть **Continuous Integration (CI) — Непрерывная интеграция**.
А если после каждой удачной сборки вы сразу ставите готовый домик на витрину, чтобы его видели люди — это **Continuous Deployment (CD) — Непрерывное развертывание**.

В программировании всё работает так же. Когда разработчики пишут код, специальный робот (в нашем случае GitLab CI/CD) берет этот код, собирает его, проверяет на ошибки (тестирует) и, если всё отлично, отправляет на рабочий сервер (развертывает).

### Основные термины GitLab CI/CD:
1. **Pipeline (Пайплайн / Конвейер)** — это весь путь вашего кода от момента загрузки до развертывания. Как конвейер на заводе.
2. **Stage (Стадия / Этап)** — шаги нашего конвейера. Обычно их три: `build` (сборка), `test` (тестирование) и `deploy` (развертывание). Стадии выполняются строго по очереди.
3. **Job (Задача / Работа)** — конкретное действие внутри стадии. Например, на стадии `test` может быть две задачи: проверка стиля кода и проверка работы программы. Задачи внутри одной стадии выполняются параллельно (одновременно).
4. **GitLab Runner (Раннер)** — это тот самый "рабочий-робот" (программа на сервере), который берет ваши задачи и выполняет их.
5. **Файл `.gitlab-ci.yml`** — это инструкция (рецепт) для GitLab Runner. В этом файле мы пишем, какие стадии нужны и какие команды на каждой стадии нужно выполнить.

## Часть 2. Практические задания

### Шаг 1: Создание проекта
1. Зайдите в свой аккаунт GitLab (если у вас его нет, необходимо зарегистрироваться).
2. Нажмите кнопку **New project** (Новый проект) -> **Create blank project** (Создать пустой проект).
3. В поле **Project name** напишите: `my-first-cicd`.
4. Оставьте видимость (Visibility Level) как **Private** или **Public**.
5. Убедитесь, что стоит галочка **Initialize repository with a README**.
6. Нажмите **Create project**.

### Шаг 2: Создание простого приложения и тестов
Мы напишем очень простую программу на языке Python, которая умеет складывать два числа, и напишем для нее тест (проверку).

1. В вашем проекте нажмите кнопку **(+) -> This directory -> New file**.
2. В поле *File name* введите `calculator.py`.
3. Вставьте в большое текстовое поле следующий код:
   ```python
   def add(x, y):
       return x + y

   if __name__ == "__main__":
       print("2 + 3 =", add(2, 3))
   ```
4. Нажмите кнопку **Commit changes** (Сохранить изменения) внизу страницы.
5. Теперь создадим файл с тестами. Снова нажмите **(+) -> New file**.
6. Назовите его `test_calculator.py` и вставьте код:
   ```python
   import calculator

   def test_add():
       assert calculator.add(2, 2) == 4
       assert calculator.add(5, 5) == 10
       print("Все тесты пройдены успешно!")

   if __name__ == "__main__":
       test_add()
   ```
7. Нажмите **Commit changes**.

### Шаг 3: Создание файла .gitlab-ci.yml
Теперь мы объясним GitLab, что нужно автоматически проверять наш калькулятор каждый раз, когда мы меняем код.

1. Нажмите **(+) -> New file**.
2. В поле *File name* введите **точно** это имя: `.gitlab-ci.yml` (точка в начале обязательна).
3. Вставьте следующий код инструкции (YAML файл очень чувствителен к пробелам, копируйте точно так, как здесь):

   ```yaml
   # 1. Выбираем базовый образ (компьютер с уже установленным Python)
   image: python:3.9

   # 2. Описываем стадии нашего конвейера (по порядку)
   stages:
     - test
     - deploy

   # 3. Создаем задачу для стадии тестирования
   run_tests:
     stage: test
     script:
       - echo "Запуск тестов калькулятора..."
       - python test_calculator.py

   # 4. Создаем задачу для стадии развертывания
   deploy_app:
     stage: deploy
     script:
       - echo "Отправка программы на сервер (имитация)..."
       - echo "Приложение успешно развернуто!"
   ```
4. Нажмите **Commit changes**.

### Шаг 4: Изучение работы Pipeline
Как только вы сохранили файл `.gitlab-ci.yml`, GitLab сразу же запустил ваш конвейер! Пойдемте на него посмотрим.

1. В левом меню GitLab наведите мышку на раздел **Build** и выберите **Pipelines**.
2. Вы увидите строку со статусом (вероятно, он будет синим кружочком `Running` или зеленым `Passed`).
3. Нажмите на значок статуса (или номер пайплайна).
4. Вы увидите визуальную схему ваших стадий: `test` и `deploy`.
5. Кликните на задачу **run_tests**. Вы попадете в консоль — это черный экран с текстом, где видно, как работает робот-runner. 
6. В самом конце логов вы должны увидеть слова: `Все тесты пройдены успешно!` и зеленую надпись `Job succeeded`.

## Часть 4. Самостоятельная работа (Задания для закрепления)

### Задание 1: Сломать Pipeline!
Настоящая польза CI/CD в том, что он не дает плохому коду пройти дальше. Давайте специально допустим ошибку.
1. Откройте файл `calculator.py` и нажмите **Edit** (Редактировать).
2. Измените функцию сложения, чтобы она работала неправильно:
   ```python
   def add(x, y):
       return x - y # Поменяли плюс на минус
   ```
3. Сохраните изменения (Commit changes).
4. Зайдите в **Build -> Pipelines**.
5. Посмотрите, что произошло. Пайплайн должен упасть (статус `Failed` с красным крестиком). 
6. Откройте логи упавшей задачи. Найдите, в каком месте тест выдал ошибку.
7. **Верните код обратно (исправьте на `x + y`)**, чтобы пайплайн снова позеленел.

### Задание 2: Добавить новую стадию сборки (Build)
В реальных проектах код перед тестированием нужно собрать.
1. Откройте файл `.gitlab-ci.yml`.
2. В раздел `stages:` добавьте стадию `build` **перед** `test`.
   ```yaml
   stages:
     - build
     - test
     - deploy
   ```
3. Ниже создайте новую задачу:
   ```yaml
   build_app:
     stage: build
     script:
       - echo "Собираем приложение..."
       - mkdir build
       - cp calculator.py build/
       - echo "Сборка завершена!"
   ```
4. Сохраните изменения и проверьте, как выглядит пайплайн теперь (в нем должно появиться 3 этапа).
