### 📝 Детализация Работ 1-3:

**Работа 1:**
1. Зарегистрироваться на GitHub.
2. Создать репозиторий `username.github.io` с файлом `README.md`. Выложить `index.html` и максимально заполнить информацию о себе (образование, опыт, навыки и т.д.). Можно использовать свои шаблоны дизайна или взять [пример с лекции](https://github.com/testilnur/testilnur.github.io/blob/main/index.html).
3. Создать репозиторий с именем `username`. Красиво оформить главную страницу профиля актуальной информацией. За основу можно взять [примеры отсюда](https://github.com/Jaidevstudio/Unique-Github-Profiles).

**Работа 2:**
1. [Скачать и установить GIT](https://git-scm.com/book/ru/v2/%D0%92%D0%B2%D0%B5%D0%B4%D0%B5%D0%BD%D0%B8%D0%B5-%D0%A3%D1%81%D1%82%D0%B0%D0%BD%D0%BE%D0%B2%D0%BA%D0%B0-Git).
2. Вызвать команду `git help status`.
3. Сделать первоначальную конфигурацию: прописать `user.name` и `user.email`.
4. Создать пустой проект и инициализировать локальный репозиторий.

**Работа 3:**
1. В локальном репозитории создать пустой исходный файл.
2. Вызвать команду `git status`.
3. Вызвать команду `git add` и сделать первый коммит.
4. Вызвать команду `git status`.
5. Добавить второй исходный файл и модифицировать первый файл.
6. Вызвать команду `git add` и сделать соответствующий коммит.
7. Вызвать команду `git log`.
8. Создать на GitHub удаленный репозиторий с файлами `.gitignore` и `README.md`.
9. Если необходимо, поменять ветку локального репозитория на `main`.
10. Привязать локальный репозиторий к удалённому: `git remote add origin ...`
11. Скачать изменения с удалённого репозитория и объединить их с локальными: `git pull origin main --allow-unrelated-histories`
12. Отправить локальные изменения в удалённый репозиторий: `git push -u origin main`
13. Проверить изменения на GitHub и удалить локальный проект (директорию).
14. Клонировать удаленный репозиторий: `git clone ...`
15. Внутри скачанного проекта (директории) проверить команду `git log`.
