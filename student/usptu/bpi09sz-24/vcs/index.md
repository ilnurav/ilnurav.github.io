### Подробное описание заданий

#### Работа 1
1. Зарегистрироваться на GitHub.
2. По аналогии с лекцией создать репозиторий `username.github.io` с `README.md` файлом, выложить `index.html` и максимально подробно заполнить информацию о себе (образование, опыт, навыки и т.д.). Можно использовать свои шаблоны и дизайн либо взять [пример из лекции](https://github.com/testilnur/testilnur.github.io/blob/main/index.html).
3. Создать репозиторий с именем `username` (специальный репозиторий профиля). Красиво оформить главную страницу профиля актуальной информацией. За основу можно взять [примеры оформления профилей](https://github.com/Jaidevstudio/Unique-Github-Profiles).
4. **Формат сдачи:** ссылка на свой профиль GitHub.

#### Работа 2
1. Скачать и установить Git, используя официальное руководство: [Введение - Установка Git](https://git-scm.com/book/ru/v2/%D0%92%D0%B2%D0%B5%D0%B4%D0%B5%D0%BD%D0%B8%D0%B5-%D0%A3%D1%81%D1%82%D0%B0%D0%BD%D0%BE%D0%B2%D0%BA%D0%B0-Git).
2. Вызвать в терминале/консоли команду `git help status`.
3. Выполнить первоначальную конфигурацию: прописать глобальные параметры `user.name` и `user.email`.
4. Создать пустой каталог для проекта и инициализировать в нем локальный репозиторий.
5. **Формат сдачи:** ссылка на скриншот(-ы) с демонстрацией всех проделанных команд в терминале.

#### Работа 3
1. В локальном репозитории создать пустой исходный файл.
2. Вызвать команду `git status`.
3. Добавить файл в индекс командой `git add` и сделать первый коммит.
4. Снова вызвать команду `git status`.
5. Добавить второй исходный файл и модифицировать первый файл.
6. Добавить изменения в индекс (`git add`) и сделать соответствующий коммит.
7. Вызвать команду `git log` для проверки истории.
8. Создать на GitHub новый удаленный репозиторий (поставив галочки для автоматического создания `.gitignore` и `README.md`).
9. При необходимости изменить имя локальной ветки на `main`.
10. Связать локальный репозиторий с удаленным: `git remote add origin <ссылка_на_репозиторий>`.
11. Скачать изменения из удаленного репозитория и объединить их с локальными:  
    `git pull origin main --allow-unrelated-histories`
12. Отправить локальные коммиты в удаленный репозиторий: `git push -u origin main`.
13. Убедиться, что все изменения отобразились на GitHub.
14. Удалить локальную директорию проекта с компьютера.
15. Клонировать удаленный репозиторий заново: `git clone <ссылка_на_репозиторий>`.
16. Зайти внутрь склонированного проекта и проверить историю изменений командой `git log`.
17. **Формат сдачи:** ссылка на созданный удаленный репозиторий на GitHub.
